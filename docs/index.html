¿Qué cambió en v4?

En v4, el SDK unifica todos los servicios bajo una sola URL: coreApiUrl. Ya no existe identityUrl porque el login, signup, auth, roles y usuarios están todos en el mismo Core API (Console Backend).

✅ Qué se agrega
  • getApplicationRoles(appId) — obtiene roles de una aplicación
  • listUsers(tenantId) — lista usuarios de un tenant
  • Validación de parámetros con error claro
  • Normalización de trailing slash en coreApiUrl

Migración v3 → v4

⚠️ Breaking change

identityUrl fue renombrado a coreApiUrl. Si usas el nombre antiguo, el constructor lanza un error claro con instrucciones.

v3 (obsoleto)v4 (actual)
identityUrl: 'https://identity.tgtone.cl'
 
coreApiUrl: 'https://dev-core.tgtone.cl/api'
Llamada v3 (con core-sdk)Llamada v4 (con auth-sdk)
core.applications.getRoles(appId)
 
authClient.getApplicationRoles(appId)
 
core.users.list(tenantId)
 
authClient.listUsers(tenantId)

Arquitectura

El auth-sdk se comunica con el Core API usando coreApiUrl como base única. Todos los endpoints:

  • Auth: login, signup, logout, refresh, exchange
  • Roles: getApplicationRoles(appId)
  • Usuarios: listUsers(tenantId)
  • Session Cache: WebSocket + REST fallback

Flujo de autenticación

El SDK maneja el flujo OAuth PKCE completo. El Core API es responsable de login, intercambio de código y validación de sesión.

OAuth PKCE flow
sequenceDiagram participant App participant CoreAPI participant User App->>CoreAPI: checkSession() CoreAPI-->>App: 401 / no session App->>CoreAPI: authorize() → redirect to login CoreAPI-->>User: Login page User->>CoreAPI: Credentials + code_verifier CoreAPI-->>App: auth code in URL App->>CoreAPI: exchange(code) → JWT CoreAPI-->>App: accessToken + refreshToken

Métodos nuevos (v4)

authClient.getApplicationRoles(appId)

Obtiene los roles disponibles para una aplicación desde el Core API.

ParámetroTipoDescripción
appIdstringUUID de la aplicación

Retorna: Promise<any[]> — array de objetos con { id, key, name, permissions }

Errores: "appId es requerido" si el parámetro está vacío. "No hay sesión activa" si no hay token.

authClient.listUsers(tenantId)

Lista usuarios de un tenant desde el Core API.

ParámetroTipoDescripción
tenantIdstringID del tenant

Retorna: Promise<any[]> — array con { userId, email, firstName, lastName, isActive, applications }

Errores: "tenantId es requerido" si el parámetro está vacío. "No hay sesión activa" si no hay token.

Manejo de errores

Novedad en v4: el constructor valida coreApiUrl al inicio y da un error claro si falta. Los métodos nuevos validan sus parámetros antes de hacer fetch.
